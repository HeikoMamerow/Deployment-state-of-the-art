4. Automatisches Deployment
	- Erklärung, (was beduetet Automatisch evl. vergleich zu Pizza oder Post einbauen)
	- Möglichkeiten,
		- Native:
			- FTP-Programme mit Sync Funktion
			- Päckchen packen -> an Server senden und auspacken (z. B. per corn)
		- Mit deployment tools
			- Begriffserklärung
					- Sind Programme mit denen Packete (Builds) erstellt werden. Entweder werden Builds dezentral,
					  also auf einem Entferntem System erzeugt oder Zentral auf dem Zielsystem.
		    - Vorrausetzung für automatische Deploys sind:
					  - Ein Versionierungssystem z.B. Git oder SVN
					  - Ein Verpackungswerkzeug (Softwaremanager) z.B. Composer für PHP Projeke.
					  - Einen Service zum ausliefern. Das können SASS Dienstleister wie Envoyer, Travis oder Bamboo sein
						oder Software wie Deployer ... (Notiz: Noch mal auf Guiseppes List schauen!) sein.
